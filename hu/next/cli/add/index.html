Version: Következő

pnpm add <pkg>

Telepít egy csomagot, valamint minden olyan csomagot, amely függősége az adott csomagnak. Alapértelmezésnél fogva, minden új csomag production függőségként kerül telepítésre.

Röviden​

ParancsJelentés
pnpm add saxMentés dependencies-be
pnpm add -D saxMentés devDependencies-be
pnpm add -O saxMentés optionalDependencies-be
pnpm add -g saxInstall package globally
pnpm add sax@nextA next címkével renderlkező verzió telepítése
pnpm add sax@3.0.0A 3.0.0 verzió telepítése

Támogatott csomag források​

Telepítés az npm bázisból​

pnpm add csomagnév a csomagnév legújabb verzióját telepíti az npm registryből.

Ha egy workspace-ben hajtják végre, a parancs először ellenőrzi, hogy valamely projekt használja-e már a megadott csomagot. Ha igen, akkor a már használt verzió lesz telepítve.

Csomagokat az alábbi parancsokkal is telepíthetsz:

  • címke használatával: pnpm add express@nightly
  • verzió használatával: pnpm add express@1.0.0
  • verzió skála használatával: pnpm add express@2 react@">=0.1.0 <0.2.0"

Telepítés a workspaceből​

Note that when adding dependencies and working within a workspace, packages will be installed from the configured sources, depending on whether or not link-workspace-packages is set, and use of the workspace: range protocol.

Telepítés helyi fájlrendszerből​

There are two ways to install from the local file system:

  1. from a tarball file (.tar, .tar.gz, or .tgz)
  2. from a directory

Példák:

pnpm add ./package.tar.gz
pnpm add ./some-directory

When you install from a directory, a symlink will be created in the current project's node_modules, so it is the same as running pnpm link.

Telepítés távoli tarballból​

The argument must be a fetchable URL starting with "http://" or "https://".

Példa:

pnpm add https://github.com/indexzero/forever/tarball/v0.5.6

Telepítés a Git tárházból​

pnpm add <git remote url>

Installs the package from the hosted Git provider, cloning it with Git. You can use a protocol for certain Git providers. For example, pnpm add github:user/repo

You may install from Git by:

  • latest commit from master: pnpm add kevva/is-positive
  • commit: pnpm add kevva/is-positive#97edff6f525f192a3f83cea1944765f769ae2678
  • branch: pnpm add kevva/is-positive#master
  • version range: pnpm add kevva/is-positive#semver:^2.0.0

Opciók​

--save-prod, -P​

A telepített csomagok a dependencies mezőbe lesznek hozzáadva.

--save-dev, -D​

A telepített csomagok a devDependencies mezőbe lesznek hozzáadva.

--save-optional, -O​

A telepített csomagok a optionalDependencies mezőbe lesznek hozzáadva.

--save-exact, -E​

A mentett függőségek pontos verzióval lesznek konfigurálva, ahelyett, hogy pnpm alapértelmezett semver-skálája legyen használva.

--save-peer​

Hozzáadva: v3.2.0

A --save-peer használatával egy vagy több csomagot ad hozzá a peerDependencies-hez és telepíti őket dev függőségekként.

--ignore-workspace-root-check, -W​

Hozzáadva: v3.6.0

Új függőség hozzáadása a gyökér workspace-projekthez tilos, kivéve, ha a --ignore-workspace-root-check vagy -W opciók vannak használva a telepítés során.

Például, pnpm add debug -W.

--global, -g​

Csomag telepítése globálisan.

--workspace​

Hozzáadva: v4.4.0

Csak akkor kerül telepítésre a függőség, ha az megtalálható a workspace-ben.

--filter <package_selector>​

Olvass többet a szűrésről.
